Inicializaste las colecciones Set a HashSet por dos razones:
- Si no inicializas las colecciones no podrás agregar entidades a esa relación porque
no puedes añadir datos a una colección null.
- Hibernate usa por defecto PersistentSet, pero esta colección siempre utiliza un tipo
de carga LAZY, provocando una LazyInicializationException que ocurre cuando intentas añadir
datos a una colección fuera de la transacción. 
En cambio HashSet lo puedes configurar como carga EAGER, evitando dicho problema

PokemonData tiene una relación @ManyToMany con MoveData. 
En PokemonData, ponemos @JsonManagedReference porque queremos que los movimientos 
asociados a un Pokémon sean serializados cuando se convierta a JSON.

MoveData tiene una relación @ManyToMany con PokemonData. 
En MoveData, ponemos @JsonBackReference porque no queremos que se serialice la lista de Pokémon para evitar la recursión. 
Esta anotación indica que, cuando serialicemos un MoveData, no debe incluirse el listado de PokemonData asociado.
